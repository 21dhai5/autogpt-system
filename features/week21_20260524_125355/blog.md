# JSON 格式化器与校验工具：从零实现一个前端实用工具

## 项目背景

本周选择的方向是 **Web 实用工具**，具体实现为一个浏览器端的 **JSON 格式化器与校验工具**。这个工具面向前端开发者、后端开发者、测试人员以及需要快速检查 JSON 数据的用户，提供以下能力：

- JSON 格式化
- JSON 压缩
- JSON 合法性校验
- 文本转 JSON 字符串字面量
- 示例数据快速加载
- 一键复制输出结果

项目完全基于原生 HTML、CSS、JavaScript 实现，无需额外框架，打开页面即可使用。

## 功能设计

### 1. JSON 格式化
用户将紧凑 JSON 粘贴到输入区域后，可以点击“格式化”按钮，程序会解析 JSON 并根据缩进设置重新生成更易读的结构化输出。

### 2. JSON 压缩
在需要将 JSON 转为紧凑格式时，可以点击“压缩”按钮，输出单行 JSON，适合接口传参和日志处理。

### 3. JSON 校验
工具会尝试解析输入内容，并反馈当前 JSON 是否合法。如果格式有误，会用中文给出失败提示。

### 4. 转义为字符串
对于需要把一段文本嵌入代码字符串中的场景，可以将内容转换为 JSON 字符串字面量。

### 5. 样例与复制
为了降低上手成本，页面内置样例 JSON，同时支持一键复制输出结果，便于开发调试。

## 技术实现

### 页面结构
`index.html` 负责定义页面布局，包括：

- 标题与功能说明
- 输入/输出双栏编辑区
- 操作按钮区
- 缩进大小选择器
- 状态信息区

### 样式设计
`styles.css` 使用了简洁明亮的卡片式布局，并通过 CSS 变量统一管理颜色。页面在桌面端使用双栏结构，在移动端自动切换为单栏，增强响应式体验。

### 核心逻辑
`app.js` 中实现了主要业务逻辑：

- 读取用户输入
- 使用 `JSON.parse()` 做解析与校验
- 使用 `JSON.stringify()` 实现格式化与压缩
- 使用 `navigator.clipboard.writeText()` 实现复制
- 根据操作结果更新状态提示

核心思路是尽量复用浏览器内置能力，保证代码轻量、易维护。

## 适用场景

这个工具适用于：

- 调试 REST API 返回结果
- 检查配置文件格式
- 处理前后端联调中的 JSON 数据
- 生成可嵌入代码的字符串内容
- 课堂教学或基础前端练习

## 项目亮点

1. **零依赖运行**：无需安装框架，直接打开 HTML 文件即可使用。
